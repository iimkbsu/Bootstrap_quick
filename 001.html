CDN(Content Delivery Network) - 컨텐츠 전송 네트워크


사용자와 가장 가까운 캐시 서버에 요청 및 응답을 받아 빠르게 컨텐츠를 사용하는 방식

CDN 방식으로 bootstrap link 태그와 script 태그 복붙 사용

HTML5 태그 class 이름 정하기

class 이름을 잘 짓는 것은 매우 중요. 특히 CSS code 작성시 효율 증대

BEM(Block Element Modifier) 방식
1. Block
>> 기능적으로 독립된 페이지 컴포넌트(요소)로 headr, content, article, navigation bar 등을 의미
>> block은 상태가 아니라 그것의 기능적 목적을 넣어야 함 (예. menu, button 등)

   ex) <div class="header"></div> : header 기능 명시
>> block은 추가적인 여백(margin, padding 등), 위치등 설정하지 않는다 (환경에 영향 받으면 안 됨)

   ex) <div class="header-margin"></div> : 사용하면 안 됨
>> BEM 방식의 경우 CSS tag나 ID 선택자를 사용하면 안 됨.

   ex) <div class="div"></div> : 사용하며 안 됨
>> block 내부에 block 존재 가능 

   ex) <div class="header">
         <div class="logo"></div>
         <div class="search-form"></div>
      </div>
>> 두 개의 단어를 - 로 연결하여 사용 가능 

   ex) <div class="main-header"></div>
2. Element
>> block 내의 하위 요소들을 의미함
>> block_element 형식으로 작성
>> 엘레멘트 이름이 두 개의 단어일 경우 block_element-name 로 작성

   ex)
   <header class="header">
      <div class="header_title"></div>
      <div class="header_icon"></div>
   </header>

   CSS
   .header {
      스타일
   }
   .header_title {
      스타일
   }
   .header_icon {
      스타일
   }
아래 CSS 코드를 사용하지 말자 

   .header .header_title {
      스타일
   }
   div.header_icon {
      스타일
   }
3. Modifier
>> 블록이나 요소의 모양 or 상태를 나타내는 이름을 사용 

   ex) size-s, theme-island, disables, focused, directrons_left-top 등
>> 각각 하나의 _ 만 사용
>> 단독으로 modifier을 사용하면 안 됨

   ex) 
   block_element-modifier
   block-name_element-name_modifier-name
   block-name_element-name_modifier-name_modifier-value
4. MIX
>> 기존 것에 기반하여 새로운 UI Components를 의미있게 만들 수 있음. 

   ex) a block과 b block이 있을 때 a 스타일을 b block 특정 요소에 적용하고 싶을 때 띄어쓰기로 다중 class를 갖게 함
   <div class="a"></div>
   <div class="b">
      <div class="b_element a"></div>
   </div>

   CSS
   .a {
      스타일
   }
